How to use our API

We have cleaned the data we retrieved from the Federal Election Commission and published this data to an API as a collection of JSONs. You can access any of these JSONs using the links in the navigation bar. Some of the JSONs have addtional functionality. We query the data from a SQLite databse, and in our Flask app we leave additional routes. In these routes, we add an extra line to our query statement based on the appropriate filter.

Employer Contributions

When looking at the JSON for employer contributions, you can specify how many results you want to see. Our default routes are "/djt_contributions" and "/hrc_contributions." You can add "/x" to the end of this route to limit the JSON to x results. For exampe, by navigating to "/djt_contributions/7," you will access a JSON which returns the top 7 contributors to DJT; by navigating to "/hrc_contributions/12," you will access a JSON which returns the top 12 contributors to HRC.

Candidate IPTs

By navigating to to "/candidate_ipt" you will access a JSON which displays the IPT for both candidates for each day of the time period we observe. By appending "/(candidate_initials)" to the end of the route, you will access a JSON which shows the IPT for only the candidate specified. For example, by navigating to "/candidate_ipt/DJT," you will access a JSON which shows the IPT each day for DJT only; HRC will not have data at this endpoint. You may navigate to "/candidate_ipt/HRC" for a JSON which shows the IPT for HRC only; the data for DJT is filtered out of this endpoint. If you append a candidate's initials to the endpoint, they must be capital. The endpoints "/candidate_ipt/djt", "/candidate_ipt/Djt", or any other permutation of capitalization will return an empty list. The same rule applies for "/candidate_ipt/HRC."
